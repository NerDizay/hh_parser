# Стандарты написания кода

## Обработка исключений

**Правило:** Нигде, кроме точки входа (`if __name__ == "__main__":`), **нельзя** использовать конструкцию `except Exception`.

### Правильный подход:
1.  **Точка входа:** Единственное место, где допустим перехват всех ошибок (`except Exception`) для логирования критического сбоя и безопасного завершения программы.
2.  **Функции и циклы:** Должны выполнять свою логику без подавления общих исключений. Если возникает ошибка внутри функции или цикла, она должна "всплывать" наверх до блока `try` в точке входа.
3.  **Специфичные исключения:** Если необходимо обработать конкретную ожидаемую ситуацию (например, отсутствие элемента на странице), используйте перехват конкретных типов исключений (например, `except NoSuchElementException`), но не общего `Exception`.

### Пример структуры:

```python
def some_function():
    # Логика без except Exception
    result = do_something() 
    return result

def main(driver):
    # Логика без except Exception
    data = some_function()
    process(data)

if __name__ == "__main__":
    try:
        driver = init_driver()
        main(driver)
    except Exception as e:
        logging.critical(f"Критическая ошибка: {e}")
    finally:
        if driver:
            driver.quit()
```

Это обеспечивает прозрачность ошибок и упрощает отладку.
